# Template Dependency Locking (P2-3-h)

This repo keeps direct dependency locks for generated templates to improve
reproducibility and supply-chain auditing.

## Scope
- `templates/python_fastapi`
- `templates/python_cli`
- `templates/python_library`

Each template keeps:
- `requirements.txt` and `requirements-dev.txt` (pinned)
- `requirements.lock` and `requirements-dev.lock` (pinned + hash comments)

## Why this matters
- **Reproducibility**: generated outputs stop floating on dependency range changes.
- **Auditability**: lock snapshots make dependency drift visible between updates.
- **Security posture**: hashes provide a tamper-evident integrity path when verification is enabled.

## Recommended install workflow
1. Prefer installing from pinned requirements for normal builds:

```bash
python -m pip install -r requirements.txt -r requirements-dev.txt
```

2. For stricter supply-chain verification (direct deps only), keep `requirements.lock`
   and verify package hashes match:

```bash
python -m pip install --require-hashes -r requirements.lock --dry-run
```

> `--dry-run` is a safety check in newer pip versions and does not mutate envs.

> Full transitive `--require-hashes` enforcement is currently best-effort in this
> repo because toolchain packages are frequently updated; use direct-dependency lock
> checks as the governance baseline and pair with periodic lock refresh.

## Lock refresh workflow
Run this from the repo root when `requirements*.txt` changes:

```bash
# 1) Refresh locks for each template
for tpl in python_cli python_fastapi python_library; do
  for kind in requirements requirements-dev; do
    req_file="templates/${tpl}/${kind}.txt"
    lock_file="templates/${tpl}/${kind}.lock"
    if [ ! -s "$req_file" ]; then
      printf '# Runtime requirements are intentionally empty\n' > "$lock_file"
      continue
    fi

    tmpdir="/tmp/autodev-lock-${tpl}-${kind}-$$"
    mkdir -p "$tmpdir"
    python3 -m pip download --no-deps --no-binary=:all: -r "$req_file" -d "$tmpdir"

    {
      echo "# Refreshed lock snapshot for $req_file"
      echo "# Generated by: python3 -m pip download --no-deps --no-binary=:all: -r $req_file"
      for f in "$tmpdir"/*.tar.gz "$tmpdir"/*.whl; do
        [ -f "$f" ] || continue
        base="$(basename "$f")"
        # crude parser for filename versions
        pkg="${base%%-*}"
        ver="$(echo "$base" | sed -E 's/^[^-]+-([0-9][^-.]*)[-.].*$/\1/')"
        h="$(python3 - <<'PY'
import hashlib,sys
path=sys.argv[1]
print(hashlib.sha256(open(path,'rb').read()).hexdigest())
PY "$f")"
        echo "${pkg}==${ver} --hash=sha256:${h}"
      done
    } > "$lock_file"

    rm -rf "$tmpdir"
  done
done
```

3. Validate that source and lock are aligned (quick check):

```bash
grep -R "^" templates/*/requirements*.lock
```
